## 1. 部品の仕分け (15分)

- 部品袋を開けたら、まず全部品を紙の上に並べて回路図のBOMと突合する
- 抵抗はカラーコードをテスターで実測確認しながらマステでラベリング
  (このキットは部品同定が一番の難所。ここを丁寧にやると後が楽)
- 極性のある部品を先に分けておく:
  **電解コンデンサ / ダイオード / LED / トランジスタ / IC** → 向き注意組

## 2. はんだ付け (60〜90分)

背の低い部品から高い部品の順。基本はこの順番:

1. 抵抗 (向きなし)
2. ダイオード (**帯の向き=カソードマーク**を基板シルクに合わせる)
3. ICソケットがあればソケット (切り欠きの向き)
4. セラミックコンデンサ (向きなし)
5. 電解コンデンサ (**長い足が+**。基板の白帯側が−)
6. インダクタ、水晶
7. トランジスタ (平らな面の向きをシルクに合わせる)
8. 半固定抵抗、タクトスイッチ、ブザー、LED (**長い足が+**)
9. ピンヘッダ (J1、電源、インターフェース)
10. **GM管クリップは最後** (背が高く邪魔になるため)

コツ:
- はんだ付けは「パッドとリードを1〜2秒温めてから、はんだを差す」。こて先にはんだを乗せて運ばない
- 1部品ごとに裏面のブリッジ(隣とショート)を目視。高圧部は特に念入りに
- リードは付けた直後に切ると散らからない

## 3. 通電前チェック (10分・省略禁止)

- [ ] 電解コンデンサの極性全数確認
- [ ] ダイオードの帯の向き全数確認
- [ ] トランジスタの向き
- [ ] 5V-GND間の導通チェック (ショートしてたら通電しない)
- [ ] **GM管はまだ付けない**

## 4. 高電圧の調整 (15分) ★このキット最大の罠

背景: 付属/GitHub文書は「テストポイントをDVMで57Vに合わせろ」と言うが、
実測した先人によるとこの手順では管に500V超がかかり、J305の定格(約400V)を
超えて管の寿命を削る。高圧源の出力インピーダンスが計算から漏れているため。

**当日の手順:**
1. GM管なしで5V給電 (ESP32の5Vピンかモバイルバッテリー+USB)
2. テストポイントにDVM(10MΩ)を当て、半固定抵抗を回して **40〜47V** に合わせる
   (換算で管に約400Vがかかる想定。57Vまで上げない)
3. 電源を切り、数分待って高圧コンデンサの放電を待つ
4. GM管をクリップに装着 (**管の向き注意: +/−の印字をシルクに合わせる**)

⚠ 高圧部(トリプラ周辺〜管のアノード)は通電中・直後は触らない。
400Vだが電流は微小なので致命的ではないものの、普通に痛い。

## 5. 動作確認 (楽しいところ)

1. 給電 → LCDが点灯し、カチ…カチ…と不定期に鳴れば成功
2. バックグラウンドで20〜30CPM程度 (数秒に1回)
3. ウランガラスを管に近づける → カチカチカチカチ (ここで歓声が上がる)
4. **鳴る間隔が不規則であることを確認する — この不規則さが本日の成果物**

## 6. ESP32接続 (30分)

1. **先にパルス電圧を測る**: VINピン(またはJ1上ピン)とGND間をテスターのDC
   レンジで確認。パルスなので正確には出ないが、プルアップ電圧が判る
   - 〜3.3V → ESP32のGPIOに直結OK
   - 5V付近 → 抵抗分圧(例: 10kΩ+20kΩ)を挟む。個体差の報告があるため要実測
2. 配線3本: 信号→GPIO4 / GND→GND / 5V→ESP32の5V(VUSB)
   - 信号は **VIN** (シルク印刷に反して、VINが出力)。またはJ1のジャンパを
     外して上ピンから取る (ジャンパを外すとカチカチ音も止まる。深夜運用向き)
3. ESP32のシリアルモニタでCPM表示を確認
4. WiFi設定を入れてkudaの /ingest へ初回POST
5. `/status` で pool_remaining が増えたら —— **第二源泉、開通**

## 7. タコパとの共存規約

- はんだ付けゾーンと調理ゾーンは物理的に分離 (フラックスの煙とたこ焼きの湯気は混ぜない)
- はんだ(特に鉛入り)を触った手でたこ焼きを掴まない。**食前に手洗い**
- 換気: はんだ付け中は窓を開けるか卓上ファンでヒュームを流す
- ウランガラスは食卓に置かない (実害はないが、行儀として)
